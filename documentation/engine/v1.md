# engine v1

The Veritate inference engine.

## what it does

- Hand-coded INT8 transformer forward + decode in C, with per-arch SIMD kernels (AVX-512 + VNNI on x86_64, NEON SDOT on ARM64).
- Byte-level (vocab=256) decoder. Input and output are raw bytes; no BPE / SentencePiece tokenizer.
- INT16 residual stream; INT8 activations at scale 32; INT8 weights with per-tensor or per-column scale.
- Optional INT4-packed weights (`prepped_b_int4_t`, `.bin` v4) for the QuaRot-rotated path.
- Optional ternary weights (BitNet b1.58, `.bin` v11 with `quant_mode=TERNARY`): 5 trits per byte on disk, decoded to `{-1, 0, +1}`-valued INT8 at load time. The runtime hot path stays INT8.
- Optional mixture-of-depths gate per block (`.bin` v6): sigmoid dot product against a learned `gate_w`; block skipped when gate fires below 0.5.
- Glass-box trace (`VRMR` v8): per-layer pre/post residuals, FFN neuron activations, attention scores, optional logit-lens projections per generated token. Drives the MRI dashboard's live activation visualization.
- Sub-millisecond per-byte decode at 16M-class shapes on M1 (~0.5 ms/byte at hidden=512, layers=6, ffn=1536); ~3 ms/byte at 80M-class.

## what it does not do

- No MoE forward at runtime yet. The v11 binary header carries `n_experts` / `router_topk` and the loader populates per-expert weights, but the FFN dispatch through the router is reserved for a follow-up. v11 binaries with `n_experts > 1` and `quant_mode != INT8` are refused at load.
- No Mamba-2 SSD path. Header `mamba2_ssd.h` exists; the prefill and decode hot paths do not route through it.

## supported `.bin` formats

| version | tag | notes |
|---|---|---|
| v3 | `VERITATE_MODEL_VERSION` | per-tensor INT8, no act_boost |
| v4 | `VERITATE_MODEL_VERSION_INT4` | INT4-packed weights, separate load path (`model_load_int4`) |
| v5 | `VERITATE_MODEL_VERSION_PERCOL` | per-output-column INT8 scales |
| v6 | `VERITATE_MODEL_VERSION_MOD` | per-col scales + per-block MoD gate |
| v8 | `VERITATE_MODEL_VERSION_NORM` | adds final `n_out` RMSNorm slot |
| v9 | `VERITATE_MODEL_VERSION_BOOST` | adds `act_boost` (residual-stream scale multiplier 1/2/4). default for `export_checkpoint` non-MoE INT8 |
| v11 | `VERITATE_MODEL_VERSION_QAT` | unified format. header carries `act_boost` + `quant_mode` (INT8 / INT4 / TERNARY) + `n_experts` + `router_topk`. covers ternary-FFN and MoE-routed checkpoints under one version. v10 was assigned twice on different branches (MoE-on-dev vs ternary-on-experimental) and was retired during the v11 unification |

## build

```bash
bash veritate_engine/v1/build/build.sh
# -> veritate_engine/v1/bin/<os>/<arch>/veritate
```

The script detects `uname -s/-m` and picks the matching kernel translation units. Apple M1 gets `-mcpu=apple-m1` (NEON + SDOT implied). x86_64 builds with AVX2 + AVX-512 + VNNI. Linux ARM64 uses `-march=armv8.2-a+dotprod`.

## run

```bash
export VERITATE_MODEL_PATH=$(pwd)/models/<name>/veritate.bin
veritate_engine/v1/bin/<os>/<arch>/veritate chat_greedy 200      # greedy decode
veritate_engine/v1/bin/<os>/<arch>/veritate chat 200             # T-sampled
veritate_engine/v1/bin/<os>/<arch>/veritate trace "prompt" out.vrmr   # VRMR trace dump
veritate_engine/v1/bin/<os>/<arch>/veritate bench 50 50          # forward + decode bench
```

Optional environment overrides:

| variable | effect |
|---|---|
| `VERITATE_ACT_BOOST=N` | override `act_boost` from the `.bin` header (N ∈ {1, 2, 4}). debugging only. |
| `VERITATE_MAX_LAYERS=N` | cap runtime layer count below the header's `layers`. early-exit experiments. |
| `VERITATE_VERIFY_DECODE` | enable `forward_decode` vs `forward` correctness check on every step. compile-time `-D` flag. |

## ternary (`.bin` v11, `quant_mode=TERNARY`) details

A v11 ternary tensor is laid out as:

```
uint8 packed[n * ceil(k/5)]                    # 5 trits per byte, base-3 encoding
int32 gamma_q24                                # round(gamma * 2^24), gamma = mean_abs(W)
```

Each packed byte holds five trit values via `byte = sum_p (trit_p + 1) * 3^p` for p in [0..4]; 3⁵ = 243 unique encodings used, 13 byte values reserved.

At load time, `load_b_ternary` decodes via a 256-byte LUT into a `{-1, 0, +1}`-valued INT8 buffer of shape `[k, n]` and runs the existing `prep_b`. The resulting `prepped_b_t` has uniform `scale_q24 = gamma_q24` (per-tensor scale, since ternary is per-tensor by definition). From the engine's hot path's perspective, the weights look like INT8 in a tight value range; the matmul + requant logic is unchanged.

The corresponding exporter is `veritate_mri.export.export_checkpoint_ternary(name, step)`. The source checkpoint must be ternary-QAT-trained: weights trained under `veritate.qat.fake_quant_weight_ternary`, with `vqat.set_qat(model, True)` and `vqat.set_quant_mode(model, vqat.QUANT_MODE_TERNARY)` set before training.

## reference numbers

Measured on Apple M1 (16 GB UMA, macOS 14.8):

| measurement | value |
|---|---|
| 16M byte-level model, v9 INT8 `.bin` | 16.0 MB |
| 16M byte-level model, v11 ternary `.bin` | 3.43 MB (4.66× smaller) |
| chat throughput, v9 INT8, 16M | ~2100 tok/s ≈ 0.48 ms/byte |
| chat throughput, v11 ternary, 16M | ~2100 tok/s ≈ 0.48 ms/byte |
| diff harness cos_dist, v11 ternary vs PyTorch ternary-QAT, all 6 layers | < 0.092 |
